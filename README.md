# Projectdocumentatie

## 1. Gebruik van Pokémon-data

Voor het opvragen van Pokémon-data maken we gebruik van PokéAPI.
Deze API wordt wekelijks door ons systeem opnieuw ingeladen, zodat de data zo up-to-date mogelijk blijft. Het is belangrijk om rekening te houden met het volgende:

- PokéAPI is gratis en één van de meest complete bronnen.
- De API is niet altijd volledig up-to-date met de allerlaatste generaties of vormen.
- Omdat PokéAPI de beste gratis optie is die bijna actueel blijft, gebruiken we deze als basis.

Na het ophalen van de data slaan we deze op in onze eigen database.
Het project gebruikt vervolgens de data uit de database, omdat dit stabieler en sneller is dan telkens opnieuw data opvragen via de API.

---

## 2. Functionaliteit van de website

De website biedt de volgende functies:

### Pokémon opzoeken
Gebruikers kunnen Pokémon opzoeken en hun informatie bekijken.
Er komt een filtersysteem waarmee je bijvoorbeeld kunt zoeken op types.

### Eigen account
Gebruikers kunnen een account aanmaken.

### Eigen Pokémon-team maken
Met een account kunnen gebruikers een eigen team samenstellen.
Er kan gekozen worden of dit team privé of openbaar is.

- Openbaar team: andere ingelogde gebruikers kunnen tips geven.
- Privé team: alleen zichtbaar voor de eigenaar.

Binnen het team kun je zien:
- Waar een Pokémon sterk tegen is
- Waar een Pokémon zwak tegen is

Dit helpt bij het plannen van battles, voorbereiden op gym leaders of bepalen welke Pokémon nuttig zijn om te vangen.

### Doel van het project
Het doel is een website maken waar gebruikers:

- Snel kunnen zien of een Pokémon de moeite waard is om te vangen
- Een team kunnen samenstellen en plannen
- Feedback kunnen krijgen van anderen
- Strategieën kunnen testen en verbeteren
